## API对接说明

### 后端API地址
- 开发环境：http://localhost:8001/api/v1
- 通过Vite代理：/api/v1

### 已封装的API模块
1. **authAPI** - 认证相关（登录、登出、获取用户信息）
2. **knowledgeAPI** - 知识点管理
3. **practiceAPI** - 实践方案生成
4. **submissionAPI** - 实践提交
5. **reviewAPI** - 审核管理

### 使用示例
```javascript
import { authAPI } from '@/api'

// 登录
const res = await authAPI.login({ username, password })

// 获取知识点列表
const list = await knowledgeAPI.getList({ category: '习概' })
```

## 状态管理

### useUserStore
```javascript
import { useUserStore } from '@/stores/user'

const userStore = useUserStore()

// 状态
userStore.token        // JWT token
userStore.user         // 用户信息

// Getters
userStore.isLoggedIn   // 是否已登录
userStore.isStudent    // 是否学生
userStore.isTeacher    // 是否教师
userStore.isAdmin      // 是否管理员

// Actions
await userStore.login(credentials)  // 登录
await userStore.logout()             // 登出
await userStore.fetchUserInfo()     // 获取用户信息
```

## 路由配置

### 路由守卫
- 自动检查登录状态
- 自动检查角色权限
- 未登录跳转到登录页
- 权限不足跳转到首页

### 路由结构
```
/login                  # 登录页
/                       # 首页
/student/*              # 学生端（需要student角色）
/teacher/*              # 教师端（需要teacher角色）
```

## 开发规范

### 1. 组件命名
- 页面组件：大驼峰命名（如 `KnowledgeSelect.vue`）
- 公共组件：大驼峰命名（如 `UserCard.vue`）

### 2. API调用
- 统一使用封装的API函数
- 使用async/await处理异步
- 错误处理由Axios拦截器统一处理

### 3. 样式
- 使用scoped样式避免污染
- 使用Element Plus主题色
- 响应式设计使用媒体查询

### 4. 代码风格
- 使用Composition API（`<script setup>`）
- 使用ref/reactive管理状态
- 使用computed计算属性

## 常见问题

### 1. npm install失败
- 检查Node.js版本（建议v18+）
- 尝试使用淘宝镜像：`npm config set registry https://registry.npmmirror.com`

### 2. 无法连接后端
- 检查后端服务是否启动
- 检查端口是否正确（8001）
- 检查Vite代理配置

### 3. 登录后跳转失败
- 检查路由配置
- 检查用户角色是否正确
- 查看浏览器控制台错误
