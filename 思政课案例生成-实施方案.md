## 核心工作流概述

### 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      【数据采集层】                          │
│   国内思政网站 → 采集器 → MySQL消息表 → ChromaDB向量库      │
└──────────────────────┬──────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────────┐
│                   【知识点匹配层】                           │
│   消息向量化 → 与知识点库撞库 → 相似度>=0.5 → 打标关联     │
└──────────────────────┬──────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────────┐
│                   【案例生成层】                             │
│   统计知识点关联消息数 → >=5条触发 → LLM聚合生成案例       │
└──────────────────────┬──────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────────────┐
│                   【存储展示层】                             │
│   案例存入MySQL → 向量化存入ChromaDB → Neo4j图谱 → 前端展示│
└─────────────────────────────────────────────────────────────┘
```

### 核心设计原则

| 原则 | 说明 |
|------|------|
| **语义匹配优先** | ❌ 不用关键词匹配<br>✅ 仅用向量相似度匹配 |
| **多消息聚合** | 一个知识点关联>=5条消息时，聚合生成1个案例 |
| **批量触发** | 积攒200条新消息后触发批量撞库（提高效率） |
| **应用场景导向** | 知识点库包含理论的实际应用场景描述 |

## 阶段2：知识点向量库构建

### 2.1 目标

从思政课教材中提取知识点，构建向量知识库，为消息匹配提供基础。

### 2.2 核心挑战

⚠️ **关键问题**：如何让向量匹配识别"可以用思政理论分析"的新闻？

**错误思路**：
- ❌ 知识点只包含理论术语（如"马克思主义"、"唯物主义"）
- ❌ 新闻必须出现这些术语才能匹配

**正确思路**：
- ✅ 知识点包含理论的**应用场景描述**（如"政府推出养老食堂解决群众急难愁盼"）
- ✅ 新闻内容与应用场景语义相似即可匹配

### 2.3 知识点数据结构设计

# -*- coding: utf-8 -*-
"""
从思政课教材中自动提取知识点
"""
import json
import asyncio
from typing import List, Dict
from backend.llm.global_llm_manager import global_llm_manager

# 提示词模板
EXTRACT_PROMPT_TEMPLATE = """
你是一位资深的思想政治理论课教师。请从以下教材章节中提取核心知识点。

【章节内容】
{chapter_text}

【提取要求】

1. 识别本章节的核心知识点（5-10个）
2. 对每个知识点，提供以下信息：

**知识点名称**：简洁明确（10-20字）
例如："以人民为中心的发展思想"、"共同富裕理论"

**理论描述**：学术定义和核心要义（150-200字）

**应用场景**：该理论在实际中的应用和体现（3-5个场景，每个30-50字）
⚠️ 重要：请用日常新闻语言描述，不要用理论术语
例如：
- "政府推出养老食堂、托幼服务等惠民政策，解决群众急难愁盼"
- "城乡教育资源均衡配置，山区孩子也能享受优质教育"
- "基层干部深入社区倾听居民诉求，改善公共服务"

**典型关键词**：与应用场景相关的常见词汇（10-15个）
⚠️ 重点包含实践领域的词汇，不仅是理论术语
例如：["民生", "惠民", "社区服务", "养老食堂", "教育公平", "医疗保障"]

【输出格式】

请严格按照JSON格式输出，不要添加任何其他文字：

```json
[
  {{
    "name": "知识点名称",
    "theory_description": "理论描述...",
    "application_scenarios": [
      "应用场景1",
      "应用场景2",
      "应用场景3"
    ],
    "typical_keywords": ["关键词1", "关键词2", "关键词3"]
  }}
]
```
"""

async def extract_knowledge_points_from_chapter(
    chapter_id: str,
    chapter_name: str,
    chapter_text: str
) -> List[Dict]:
    """
    从单个章节提取知识点
    """
    prompt = EXTRACT_PROMPT_TEMPLATE.format(chapter_text=chapter_text)
    
    # 调用LLM
    response = await global_llm_manager.chat_client.chat(
        messages=[
            {
                "role": "system", 
                "content": "你是一位资深的思想政治理论课教师，擅长提取和总结知识点。"
            },
            {"role": "user", "content": prompt}
        ],
        temperature=0.3,  # 较低温度保证稳定输出
        max_tokens=3000
    )
    
    # 解析JSON
    content = response.choices[0].message.content
    
    # 提取JSON部分（可能包含```json```标记）
    import re
    json_match = re.search(r'```json\s*(\[.*?\])\s*```', content, re.DOTALL)
    if json_match:
        content = json_match.group(1)
    
    knowledge_points = json.loads(content)
    
    # 添加章节信息和生成embedding_text
    for kp in knowledge_points:
        kp['chapter'] = chapter_name
        kp['chapter_id'] = chapter_id
        
        # 生成embedding_text（理论+关键词+场景）
        kp['embedding_text'] = f"{kp['name']} {kp['theory_description'][:100]} {' '.join(kp['typical_keywords'])} {' '.join(kp['application_scenarios'])}"
    
    return knowledge_points

async def extract_all_knowledge_points(textbook_file: str = "backend/data/textbook_chapters.json"):
    """
    提取整本教材的知识点
    """
    # 读取教材章节
    with open(textbook_file, 'r', encoding='utf-8') as f:
        textbook = json.load(f)
    
    all_knowledge_points = []
    
    for chapter in textbook:
        print(f"\n处理章节: {chapter['chapter_name']}")
        
        # 拼接章节文本
        chapter_text = "\n\n".join([
            f"【{section['section_name']}】\n{section['text']}"
            for section in chapter['sections']
        ])
        
        # 提取知识点
        try:
            kps = await extract_knowledge_points_from_chapter(
                chapter_id=chapter['chapter_id'],
                chapter_name=chapter['chapter_name'],
                chapter_text=chapter_text
            )
            
            print(f"  提取到 {len(kps)} 个知识点")
            all_knowledge_points.extend(kps)
            
            # 延迟避免API限流
            await asyncio.sleep(2)
            
        except Exception as e:
            print(f"  提取失败: {e}")
            continue
    
    # 保存结果
    output_file = "backend/data/knowledge_points.json"
    with open(output_file, 'w', encoding='utf-8') as f:
        json.dump(all_knowledge_points, f, ensure_ascii=False, indent=2)
    
    print(f"\n✅ 成功提取 {len(all_knowledge_points)} 个知识点")
    print(f"📁 保存到: {output_file}")
    
    return all_knowledge_points

if __name__ == "__main__":
    asyncio.run(extract_all_knowledge_points())

#### Step 2.3: 向量化并存入ChromaDB

**脚本**: `backend/scripts/build_knowledge_point_vectors.py`

```python
# -*- coding: utf-8 -*-
"""
构建知识点向量库
"""
import json
import uuid
from backend.llm.global_llm_manager import global_llm_manager
from backend.storage.chromadb_storage import chromadb_storage

async def build_knowledge_point_vectors():
    """
    将知识点向量化并存入ChromaDB
    """
    # 读取知识点
    with open('backend/data/knowledge_points.json', 'r', encoding='utf-8') as f:
        knowledge_points = json.load(f)
    
    collection_name = "iptc_knowledge_points"
    
    print(f"开始向量化 {len(knowledge_points)} 个知识点...")
    
    for i, kp in enumerate(knowledge_points, 1):
        try:
            # 生成唯一ID
            kp_id = kp.get('id', str(uuid.uuid4()))
            
            # 使用embedding_text进行向量化
            embedding_text = kp['embedding_text']
            
            # 调用Embedding模型
            embedding = await global_llm_manager.embedding_client.embed(embedding_text)
            
            # 存入ChromaDB
            chromadb_storage.add(
                collection_name=collection_name,
                id=kp_id,
                embedding=embedding,
                document=kp['theory_description'],  # 存储理论描述作为文档
                metadata={
                    "name": kp['name'],
                    "chapter": kp['chapter'],
                    "chapter_id": kp['chapter_id'],
                    "application_scenarios": json.dumps(kp['application_scenarios'], ensure_ascii=False),
                    "typical_keywords": json.dumps(kp['typical_keywords'], ensure_ascii=False)
                }

        except Exception as e:

## 阶段3：智能案例生成流程

### 3.1 目标

实现批量撞库机制：积攒200条新消息 → 向量匹配知识点 → 统计关联数 → >=5条触发案例生成。

### 3.2 核心流程图

```
新消息持续采集
    ↓
每小时检查未处理消息数
    ↓
积攒>=200条？ → No → 继续等待
    ↓ Yes
【批量撞库任务】
    ├─ Step 1: 读取200条新消息
    ├─ Step 2: 逐条向量化
    ├─ Step 3: 与知识点库检索（Top 3）
    ├─ Step 4: 筛选相似度>=0.5的匹配
    ├─ Step 5: 记录到iptc_message_knowledge_relations表
    └─ Step 6: 统计每个知识点的关联消息数
    ↓
某知识点关联消息>=5条？ → No → 继续积攒
    ↓ Yes
【案例生成任务】
    ├─ Step 1: 提取该知识点的所有关联消息
    ├─ Step 2: 聚合消息内容
    ├─ Step 3: 调用LLM生成案例（使用提示词模板）
    ├─ Step 4: 保存到iptc_cases表
    ├─ Step 5: 向量化并存入ChromaDB
    └─ Step 6: 同步到Neo4j图谱
```

### 3.4 核心实现

#### 脚本: `backend/scripts/batch_match_cases.py`

```python
# -*- coding: utf-8 -*-
"""
批量撞库案例生成服务
"""
import uuid
import json
import asyncio
from datetime import datetime
from typing import List, Dict
from collections import defaultdict

from backend.database.connection import create_session
from backend.database.entities import (
    PeopleTheoryMessage,
    QstheoryMessage,
    GmwTheoryMessage,
    CssnMessage
)
from backend.storage.chromadb_storage import chromadb_storage
from backend.llm.global_llm_manager import global_llm_manager
from sqlalchemy import text

class BatchMatchCasesService:
    """批量撞库案例生成服务"""
    
    # 配置参数
    BATCH_SIZE = 200                    # 批量处理消息数
    SIMILARITY_THRESHOLD = 0.5          # 相似度阈值
    CASE_GENERATION_THRESHOLD = 5       # 案例生成阈值（消息数）
    
    # 国内思政消息源表名
    SOURCE_TABLES = [
        "mp_people_theory_messages",
        "mp_qstheory_messages",
        "mp_gmw_theory_messages",
        "mp_cssn_messages"
    ]
    
    def __init__(self):
        self.logger = logging.getLogger(__name__)
    
    async def run_batch_match(self):
        """
        执行批量撞库任务
        """
        self.logger.info("="*60)
        self.logger.info("开始批量撞库任务")
        self.logger.info("="*60)
        
        # ========== Step 1: 获取待处理消息 ==========
        messages = self._get_unprocessed_messages()
        
        if len(messages) < self.BATCH_SIZE:
            self.logger.info(f"当前未处理消息数: {len(messages)}/{self.BATCH_SIZE}，等待积攒")
            return
        
        self.logger.info(f"获取到 {len(messages)} 条待处理消息，开始撞库...")
        
        # ========== Step 2: 向量撞库匹配知识点 ==========
        matches = await self._match_knowledge_points(messages)
        self.logger.info(f"匹配到 {len(matches)} 条消息-知识点关联")
        
        # ========== Step 3: 保存关联关系到数据库 ==========
        self._save_relations(matches)
        
        # ========== Step 4: 统计知识点关联数 ==========
        kp_stats = self._calculate_kp_statistics()
        self.logger.info(f"统计到 {len(kp_stats)} 个知识点有关联消息")
        
        # ========== Step 5: 触发案例生成 ==========
        generated_cases = await self._generate_cases(kp_stats)
        
        self.logger.info("="*60)
        self.logger.info(f"✅ 批量撞库任务完成")
        self.logger.info(f"   - 处理消息: {len(messages)} 条")
        self.logger.info(f"   - 匹配关联: {len(matches)} 条")
        self.logger.info(f"   - 生成案例: {len(generated_cases)} 个")
        self.logger.info("="*60)
        
        return {
            "total_messages": len(messages),
            "matched_pairs": len(matches),
            "generated_cases": len(generated_cases)
        }
    
    def _get_unprocessed_messages(self) -> List[Dict]:
        """
        获取未处理的消息
        """
        messages = []
        
        with create_session() as db:
            for table_name in self.SOURCE_TABLES:
                # 查询该表中未被打标的消息（不在relations表中）
                query = text(f"""
                    SELECT id, title, content, summary, published_at, url, '{table_name}' as source_table
                    FROM {table_name}
                    WHERE id NOT IN (
                        SELECT message_id FROM iptc_message_knowledge_relations
                        WHERE source_table = '{table_name}'
                    )
                    ORDER BY published_at DESC
                    LIMIT :limit
                """)
                
                results = db.execute(query, {"limit": self.BATCH_SIZE // len(self.SOURCE_TABLES)})
                
                for row in results:
                    messages.append({
                        "id": row.id,
                        "title": row.title,
                        "content": row.content,
                        "summary": row.summary,
                        "published_at": row.published_at,
                        "url": row.url,
                        "source_table": row.source_table
                    })
        
        return messages[:self.BATCH_SIZE]
    
    async def _match_knowledge_points(self, messages: List[Dict]) -> List[Dict]:
        """
        向量撞库匹配知识点
        """
        matches = []
        
        for i, msg in enumerate(messages, 1):
            try:
                # 构造查询文本
                query_text = f"{msg['title']} {msg['content'][:1000]}"
                
                # 向量检索Top 3知识点
                results = chromadb_storage.search(
                    collection_name="iptc_knowledge_points",
                    query_text=query_text,
                    n_results=3
                )
                
                # 筛选相似度>=0.6的知识点
                for result in results:
                    if result['similarity'] >= self.SIMILARITY_THRESHOLD:
                        matches.append({
                            "message_id": msg['id'],
                            "source_table": msg['source_table'],
                            "knowledge_point_id": result['id'],
                            "knowledge_point_name": result['metadata']['name'],
                            "similarity": result['similarity']
                        })
                
                if i % 50 == 0:
                    self.logger.info(f"  进度: {i}/{len(messages)}")
                
            except Exception as e:
                self.logger.error(f"匹配失败 {msg['id']}: {e}")
        
        return matches
    
    def _save_relations(self, matches: List[Dict]):
        """
        保存消息-知识点关联关系
        """
        with create_session() as db:
            for match in matches:
                # 插入或更新
                query = text("""
                    INSERT INTO iptc_message_knowledge_relations 
                    (id, message_id, source_table, knowledge_point_id, knowledge_point_name, similarity_score)
                    VALUES (:id, :message_id, :source_table, :kp_id, :kp_name, :similarity)
                    ON DUPLICATE KEY UPDATE similarity_score = :similarity
                """)
                
                db.execute(query, {
                    "id": str(uuid.uuid4()),
                    "message_id": match['message_id'],
                    "source_table": match['source_table'],
                    "kp_id": match['knowledge_point_id'],
                    "kp_name": match['knowledge_point_name'],
                    "similarity": match['similarity']
                })
            
            db.commit()
        
        self.logger.info(f"  ✅ 保存 {len(matches)} 条关联关系")
    
    def _calculate_kp_statistics(self) -> Dict[str, Dict]:
        """
        统计每个知识点的关联消息数
        """
        with create_session() as db:
            query = text("""
                SELECT 
                    knowledge_point_id,
                    knowledge_point_name,
                    COUNT(*) as message_count
                FROM iptc_message_knowledge_relations
                WHERE knowledge_point_id NOT IN (
                    SELECT knowledge_point_id FROM iptc_knowledge_point_stats
                    WHERE case_generated = 1
                )
                GROUP BY knowledge_point_id, knowledge_point_name
                HAVING message_count >= :threshold
            """)
            
            results = db.execute(query, {"threshold": self.CASE_GENERATION_THRESHOLD})
            
            kp_stats = {}
            for row in results:
                kp_stats[row.knowledge_point_id] = {
                    "name": row.knowledge_point_name,
                    "count": row.message_count
                }
        
        return kp_stats
    
    async def _generate_cases(self, kp_stats: Dict[str, Dict]) -> List[Dict]:
        """
        生成案例
        """
        generated_cases = []
        
        for kp_id, stat in kp_stats.items():
            self.logger.info(f"\n生成案例: {stat['name']} (关联消息: {stat['count']}条)")
            
            try:
                # 获取该知识点的所有关联消息
                messages = self._get_related_messages(kp_id)
                
                # 调用LLM生成案例
                case = await self._generate_case_for_knowledge_point(
                    knowledge_point_id=kp_id,
                    knowledge_point_name=stat['name'],
                    related_messages=messages
                )
                
                # 保存案例
                self._save_case(case)
                
                # 标记知识点已生成案例
                self._mark_case_generated(kp_id, stat['name'])
                
                generated_cases.append(case)
                
                self.logger.info(f"  ✅ 案例生成成功: {case['title']}")
                
            except Exception as e:
                self.logger.error(f"  ❌ 案例生成失败: {e}")
        
        return generated_cases
    
    def _get_related_messages(self, kp_id: str) -> List[Dict]:
        """
        获取知识点关联的所有消息
        """
        messages = []
        
        with create_session() as db:
            query = text("""
                SELECT 
                    r.message_id,
                    r.source_table,
                    r.similarity_score
                FROM iptc_message_knowledge_relations r
                WHERE r.knowledge_point_id = :kp_id
                ORDER BY r.similarity_score DESC
            """)
            
            results = db.execute(query, {"kp_id": kp_id})
            
            for row in results:
                # 从对应的消息表查询完整消息
                msg_query = text(f"""
                    SELECT id, title, content, summary, provider, published_at, url
                    FROM {row.source_table}
                    WHERE id = :msg_id
                """)
                
                msg_result = db.execute(msg_query, {"msg_id": row.message_id}).first()
                
                if msg_result:
                    messages.append({
                        "id": msg_result.id,
                        "title": msg_result.title,
                        "content": msg_result.content,
                        "summary": msg_result.summary,
                        "provider": msg_result.provider,
                        "published_at": msg_result.published_at,
                        "url": msg_result.url,
                        "similarity": row.similarity_score
                    })
        
        return messages
    
    async def _generate_case_for_knowledge_point(
        self,
        knowledge_point_id: str,
        knowledge_point_name: str,
        related_messages: List[Dict]
    ) -> Dict:
        """
        为单个知识点生成案例（聚合多条消息）
        """
        # 1. 聚合消息内容
        aggregated_content = self._aggregate_messages(related_messages)
        
        # 2. 读取提示词模板
        template_path = r"D:\AI-Empowered IPTC Website\新闻转案例提示词.md"
        with open(template_path, 'r', encoding='utf-8') as f:
            prompt_template = f.read()
        
        # 3. 填充模板
        prompt = prompt_template.format(
            news_content=aggregated_content['content'],
            news_title=aggregated_content['title'],
            news_source="多源综合",
            published_at=related_messages[0]['published_at'].strftime("%Y-%m-%d"),
            url="多条消息聚合生成",
            related_knowledge_points=f"- {knowledge_point_name}"
        )
        
        # 4. 调用LLM生成案例
        response = await global_llm_manager.chat_client.chat(
            messages=[
                {
                    "role": "system",
                    "content": "你是一位资深的思想政治理论课教师和教学案例编写专家。"
                },
                {"role": "user", "content": prompt}
            ],
            temperature=0.7,
            max_tokens=4000
        )
        
        case_content = response.choices[0].message.content
        
        # 5. 提取案例元数据
        import re
        title_match = re.search(r'^#\s+(.+)$', case_content, re.MULTILINE)
        title = title_match.group(1) if title_match else f"{knowledge_point_name}相关案例"
        
        summary_match = re.search(r'##\s+核心阅读\s+(.+?)(?=\n##)', case_content, re.DOTALL)
        summary = summary_match.group(1).strip() if summary_match else case_content[:500]
        
        # 6. 构造案例对象
        case = {
            "id": str(uuid.uuid4()),
            "title": title,
            "content": case_content,
            "summary": summary,
            "source_url": related_messages[0]['url'],
            "tags": knowledge_point_name,
            "related_knowledge_points": [{
                "id": knowledge_point_id,
                "name": knowledge_point_name
            }],
            "source_message_ids": [msg['id'] for msg in related_messages],
            "published_at": related_messages[0]['published_at'],
            "created_at": datetime.now()
        }
        
        return case
    
    def _aggregate_messages(self, messages: List[Dict]) -> Dict:
        """
        聚合多条消息的内容
        """
        # 策略：取第一条标题 + 所有内容摘要
        title = messages[0]['title']
        
        content_parts = []
        for i, msg in enumerate(messages, 1):
            content_parts.append(f"""
【消息{i}】来源: {msg['provider']} | 时间: {msg['published_at'].strftime("%Y-%m-%d")}
标题: {msg['title']}
内容: {msg['content'][:800] if msg['content'] else msg['summary']}...
""")
        
        content = "\n\n".join(content_parts)
        
        return {"title": title, "content": content}
    
    def _save_case(self, case: Dict):
        """
        保存案例到数据库
        """
        with create_session() as db:
            query = text("""
                INSERT INTO iptc_cases 
                (id, title, content, summary, source_url, tags, 
                 related_knowledge_points, source_message_ids, published_at, created_at)
                VALUES 
                (:id, :title, :content, :summary, :source_url, :tags,
                 :related_kps, :source_msg_ids, :published_at, :created_at)
            """)
            
            db.execute(query, {
                "id": case['id'],
                "title": case['title'],
                "content": case['content'],
                "summary": case['summary'],
                "source_url": case['source_url'],
                "tags": case['tags'],
                "related_kps": json.dumps(case['related_knowledge_points'], ensure_ascii=False),
                "source_msg_ids": json.dumps(case['source_message_ids'], ensure_ascii=False),
                "published_at": case['published_at'],
                "created_at": case['created_at']
            })
            
            db.commit()
    
    def _mark_case_generated(self, kp_id: str, kp_name: str):
        """
        标记知识点已生成案例
        """
        with create_session() as db:
            query = text("""
                INSERT INTO iptc_knowledge_point_stats
                (id, knowledge_point_id, knowledge_point_name, case_generated, last_matched_at)
                VALUES
                (:id, :kp_id, :kp_name, 1, NOW())
                ON DUPLICATE KEY UPDATE
                case_generated = 1, last_matched_at = NOW()
            """)
            
            db.execute(query, {
                "id": str(uuid.uuid4()),
                "kp_id": kp_id,
                "kp_name": kp_name
            })
            
            db.commit()


async def main():
    """主函数"""
    service = BatchMatchCasesService()
    await service.run_batch_match()

if __name__ == "__main__":
    import logging
    logging.basicConfig(level=logging.INFO)
    asyncio.run(main())

## 关键技术细节

### 向量相似度阈值调优

| 阈值 | 匹配效果 | 适用场景 |
|------|---------|---------|
| 0.5 | 宽松，容易误匹配 | 初期测试，扩大召回 |
| **0.6** | **平衡，推荐值** | **生产环境** |
| 0.7 | 严格，可能漏匹配 | 要求高精度 |

### 案例生成触发阈值

| 阈值 | 说明 |
|------|------|
| **7条** | **推荐值**，保证案例有足够素材 |
| 5条 | 最低值，内容可能不够丰富 |
| 10条 | 高质量，但触发频率低 |

## 附录：快速命令参考

### 创建数据库表

```bash
mysql -u root -p message_platform < backend/scripts/create_tables.sql
```

### 提取知识点

```bash
conda activate personal_agent
cd message_platform
python backend/scripts/extract_knowledge_points.py
```

### 构建向量库

```bash
python backend/scripts/build_knowledge_point_vectors.py
```

### 测试匹配效果

```bash
python backend/scripts/test_knowledge_point_matching.py
```

### 手动触发批量撞库

```bash
python backend/scripts/batch_match_cases.py
```

### 启动定时任务

```bash
celery -A backend.tasks worker --loglevel=info --pool=solo -Q default,collector
celery -A backend.tasks beat --loglevel=info
```
