# 配置文件目录说明

## 目录作用

config目录包含GraphRAG的配置文件和提示词模板，用于管理Neo4j连接、LLM配置、实体提取规则等。

## 文件列表

- `default_config.yaml` - 默认配置模板
- `prompts/` - 提示词模板目录
  - `entity_extraction.txt` - 实体提取提示词

## 技术说明

### 配置文件格式

使用YAML格式，支持环境变量替换：
- `${VAR}` - 必需的环境变量
- `${VAR:default}` - 可选的环境变量，提供默认值

### 配置项说明

**Neo4j配置**：
- `uri` - 连接URI（默认bolt://localhost:7687）
- `user` - 用户名（默认neo4j）
- `password` - 密码（必需）
- `database` - 数据库名称（默认neo4j）
- `max_connection_pool_size` - 连接池大小（默认50）
- `connection_timeout` - 连接超时（默认30秒）

**LLM配置**：
- `provider` - 提供商（openai/anthropic/custom）
- `model` - 模型名称
- `api_key` - API密钥（必需）
- `base_url` - API基础URL（可选）
- `temperature` - 温度参数（默认0.3）
- `max_tokens` - 最大token数（默认2000）

**实体提取配置**：
- `max_chunk_chars` - 单个文本块最大字符数（默认20000）
- `entity_types` - 支持的实体类型列表
- `relation_types` - 支持的关系类型列表

**实体归一化配置**：
- `similarity_threshold` - 相似度阈值（默认0.9）
- `enable_auto_merge` - 是否自动合并（默认true）

## 使用示例

### 加载配置

```python
from graphrag.utils.config_loader import ConfigLoader

config = ConfigLoader.load_config("config/default_config.yaml")
print(config['neo4j']['uri'])
```

### 使用环境变量

```bash
# 设置环境变量
export NEO4J_PASSWORD=your_password
export LLM_API_KEY=your_api_key

# 运行程序
python your_script.py
```

### 自定义配置

创建新的配置文件：

```yaml
# my_config.yaml
neo4j:
  uri: bolt://my-server:7687
  user: admin
  password: ${MY_NEO4J_PASSWORD}

llm:
  provider: anthropic
  model: claude-3-5-sonnet-20241022
  api_key: ${MY_ANTHROPIC_KEY}
```

## 扩展方式

### 添加新配置项

在`default_config.yaml`中添加新配置：

```yaml
# 新增配置项
your_feature:
  enabled: true
  option1: value1
  option2: value2
```

### 修改提示词

编辑`prompts/entity_extraction.txt`：

```
你是一个专业的知识图谱构建助手。

## 新增规则
- 你的新规则1
- 你的新规则2
```

## 注意事项

### 敏感信息管理

- 不要在配置文件中硬编码密码
- 使用环境变量管理敏感信息
- 使用`.env`文件管理本地开发环境
- 不要将`.env`文件提交到版本控制

### 配置验证

- 启动前验证配置完整性
- 使用`ConfigLoader.validate_config()`
- 提供清晰的错误提示

### 环境变量优先级

1. 系统环境变量
2. `.env`文件
3. 配置文件中的默认值

### 提示词优化

- 提示词直接影响实体提取质量
- 根据实际效果调整提示词
- 提供清晰的示例和规则
- 避免过于复杂的指令
