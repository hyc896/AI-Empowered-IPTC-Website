# 使用示例目录说明

## 目录作用

examples目录包含GraphRAG工具包的使用示例，演示不同场景下的使用方法。

## 文件列表

- `basic_usage.py` - 基础使用示例，演示完整的实体提取和图谱构建流程
- `with_message_platform.py` - 与message_platform集成示例，演示如何复用现有配置和LLM
- `custom_llm.py` - 自定义LLM客户端示例，演示如何实现自定义LLM

## 技术说明

### 基础使用示例（basic_usage.py）

**演示内容**：
- 加载配置文件
- 初始化Neo4j存储
- 初始化LLM客户端（OpenAI）
- 初始化实体提取器和图谱构建器
- 从文本中提取实体和关系
- 构建知识图谱
- 查询实体

**适用场景**：
- 独立使用GraphRAG
- 学习基本使用流程
- 快速上手

### message_platform集成示例（with_message_platform.py）

**演示内容**：
- 复用message_platform的配置
- 复用message_platform的LLM客户端
- 从message_platform数据库读取消息
- 批量处理消息并构建图谱

**适用场景**：
- 与message_platform集成
- 复用现有基础设施
- 批量处理历史数据

### 自定义LLM示例（custom_llm.py）

**演示内容**：
- 实现自定义LLM客户端
- 继承BaseLLMClient基类
- 实现generate_async和generate方法
- 使用自定义LLM进行实体提取

**适用场景**：
- 使用本地模型
- 使用第三方API
- 自定义LLM行为

## 使用示例

### 运行基础示例

```bash
# 设置环境变量
export NEO4J_PASSWORD=your_password
export LLM_API_KEY=your_api_key

# 运行示例
cd GraphRAG
python examples/basic_usage.py
```

### 运行集成示例

```bash
# 确保message_platform在正确路径
cd GraphRAG
python examples/with_message_platform.py
```

### 运行自定义LLM示例

```bash
cd GraphRAG
python examples/custom_llm.py
```

### 添加错误处理

在示例中添加错误处理：

```python
try:
    result = await extractor.extract_entities(text)
except Exception as e:
    print(f"实体提取失败: {e}")
    return
```

### 添加日志输出

在示例中添加详细日志：

## 注意事项

### 环境准备

- 确保Neo4j服务运行
- 设置正确的环境变量
- 安装所有依赖

### 路径问题

- 示例使用相对路径
- 确保在正确的目录运行
- 使用`sys.path.insert()`添加父目录

### 资源清理

- 示例结束后关闭连接
- 避免资源泄漏
- 使用try-finally确保清理

### 错误处理

- 捕获并处理异常
- 提供清晰的错误信息
- 记录详细的日志

### 性能考虑

- 示例使用小数据集
- 生产环境需要批量处理
- 注意LLM API速率限制
