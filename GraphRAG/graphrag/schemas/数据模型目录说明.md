# 数据模型目录说明

## 目录作用

schemas目录使用Pydantic定义GraphRAG的数据模型，提供类型验证、数据序列化和文档生成功能。

## 文件列表

- `__init__.py` - 模块初始化文件
- `entity.py` - 实体模型定义
- `relation.py` - 关系模型定义
- `graph.py` - 图数据模型定义

## 技术说明

### Pydantic数据模型

使用Pydantic v2进行数据验证和序列化：
- 自动类型检查
- JSON序列化/反序列化
- 数据验证
- 自动生成文档

### 实体模型（entity.py）

**字段说明**：
- `name` - 实体名称（必需）
- `type` - 实体类型（必需）
- `aliases` - 别名列表（可选）
- `attributes` - 属性字典（可选）
- `mention_count` - 提及次数（默认1）

### 关系模型（relation.py）

**字段说明**：
- `source` - 源实体名称（必需）
- `target` - 目标实体名称（必需）
- `type` - 关系类型（必需）
- `properties` - 关系属性（可选）

### 图数据模型（graph.py）

**字段说明**：
- `nodes` - 节点列表
- `links` - 边列表
- `categories` - 分类列表

## 使用示例

### 创建实体

```python
from graphrag.schemas.entity import Entity

entity = Entity(
    name="OpenAI",
    type="Company",
    aliases=["Open AI"],
    attributes={"country": "美国", "industry": "人工智能"}
)

print(entity.model_dump())
```

### 创建关系

```python
from graphrag.schemas.relation import Relation

relation = Relation(
    source="Sam Altman",
    target="OpenAI",
    type="WORKS_AT",
    properties={"role": "CEO"}
)

print(relation.model_dump())
```

### 创建图数据

```python
from graphrag.schemas.graph import GraphData

graph = GraphData(
    nodes=[{"id": "1", "name": "OpenAI"}],
    links=[{"source": "1", "target": "2"}],
    categories=[{"name": "Company"}]
)

print(graph.model_dump_json())
```

## 扩展方式

### 添加新字段

在模型类中添加新字段：

```python
from pydantic import BaseModel, Field

class Entity(BaseModel):
    name: str
    type: str
    your_new_field: str = Field(default="", description="新字段")
```

### 添加验证规则

使用Pydantic的验证器：

```python
from pydantic import field_validator

class Entity(BaseModel):
    name: str

    @field_validator('name')
    def name_must_not_be_empty(cls, v):
        if not v.strip():
            raise ValueError('名称不能为空')
        return v
```

## 注意事项

### 类型安全

- 使用类型注解确保类型安全
- Pydantic自动进行类型转换
- 验证失败会抛出ValidationError

### 序列化

- `model_dump()` - 转换为字典
- `model_dump_json()` - 转换为JSON字符串
- `model_validate()` - 从字典创建实例

### 性能考虑

- Pydantic v2性能优于v1
- 避免频繁创建模型实例
- 使用`model_construct()`跳过验证
