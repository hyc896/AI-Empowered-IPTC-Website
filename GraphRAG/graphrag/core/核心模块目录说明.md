# 核心模块目录说明

## 目录作用

core目录包含GraphRAG的核心功能模块，负责Neo4j存储、实体提取、图谱构建和查询。这些模块是GraphRAG工具包的核心业务逻辑。

## 文件列表

- `__init__.py` - 模块初始化文件
- `storage.py` - Neo4j存储层，提供图数据库连接和CRUD操作
- `entity_extractor.py` - 实体提取器，使用LLM从文本中提取实体和关系
- `graph_builder.py` - 图谱构建器，将实体和关系写入Neo4j
- `graph_query.py` - 图谱查询器，封装常用查询模式（待实现）

## 技术说明

### Neo4j存储层（storage.py）

**核心功能**：
- Driver实例管理（非单例模式，支持多实例）
- 连接验证（Fail Fast原则）
- 读写事务封装
- 异步操作支持（ThreadPoolExecutor）

**关键方法**：
- `initialize(config)` - 初始化驱动，验证连接
- `execute_read(query, params)` - 执行读事务
- `execute_write(query, params)` - 执行写事务
- `execute_read_async(query, params)` - 异步读事务
- `execute_write_async(query, params)` - 异步写事务
- `close()` - 关闭驱动和线程池

### 实体提取器（entity_extractor.py）

**核心功能**：
- 支持8种实体类型（Company、Person、Technology、Product、Organization、Event、Concept、Location）
- 支持7种关系类型（WORKS_AT、INVESTS_IN、DEVELOPS、COMPETES_WITH、PARTNERS_WITH、BASED_ON、LOCATED_IN）
- 长文本分块处理（默认20000字符）
- 幻觉检测机制（检测模板占位符、引导式回复、元描述）
- JSON格式验证

**关键方法**：
- `extract_entities(text, language)` - 提取实体和关系
- `_extract_entities_single(text, language)` - 单块提取
- `_extract_entities_chunked(text, language)` - 分块提取
- `_is_hallucination(response)` - 幻觉检测
- `_validate_extraction_result(result)` - 结果验证

### 图谱构建器（graph_builder.py）

**核心功能**：
- 创建Document节点（抽象的文档节点）
- 创建或更新Entity节点（使用MERGE去重）
- 创建MENTIONS关系（Document → Entity）
- 创建Entity间关系
- 删除文档及其关系

**关键方法**：
- `add_document_with_entities(doc_id, doc_data, entities, relations)` - 添加文档及其实体
- `_create_document_node(doc_id, doc_data)` - 创建文档节点
- `_merge_entity_node(entity)` - 创建或更新实体节点
- `_create_mentions_relation(doc_id, entity_name)` - 创建提及关系
- `_create_entity_relation(relation)` - 创建实体间关系
- `delete_document_graph(doc_id)` - 删除文档图谱

## 使用示例

### 存储层使用

```python
from graphrag.core.storage import Neo4jStorage

# 初始化
storage = Neo4jStorage()
storage.initialize({
    'uri': 'bolt://localhost:7687',
    'user': 'neo4j',
    'password': 'your_password',
    'database': 'neo4j'
})

# 同步查询
results = storage.execute_read("MATCH (n) RETURN n LIMIT 10")

# 异步查询
results = await storage.execute_read_async("MATCH (n) RETURN n LIMIT 10")

# 关闭
storage.close()
```

### 实体提取使用

```python
from graphrag.core.entity_extractor import EntityExtractor
from graphrag.llm.openai_client import OpenAIClient

# 初始化
llm_client = OpenAIClient(api_key="your_key")
extractor = EntityExtractor(llm_client)

# 提取实体
text = "OpenAI发布了GPT-4模型。Sam Altman是OpenAI的CEO。"
result = await extractor.extract_entities(text)

print(f"实体: {result['entities']}")
print(f"关系: {result['relations']}")
```

### 图谱构建使用

```python
from graphrag.core.graph_builder import GraphBuilder

# 初始化
builder = GraphBuilder(storage)

# 构建图谱
doc_data = {
    "title": "新闻标题",
    "content": "新闻内容",
    "url": "https://example.com/news/1",
    "published_at": "2025-01-01T00:00:00",
    "crawled_at": "2025-01-01T00:00:00"
}

stats = await builder.add_document_with_entities(
    doc_id="doc_001",
    doc_data=doc_data,
    entities=result['entities'],
    relations=result['relations']
)

print(f"创建实体: {stats['entities_created']}")
print(f"创建关系: {stats['relations_created']}")
```

## 扩展方式

### 添加新的实体类型

1. 修改`entity_extractor.py`中的`ENTITY_TYPES`列表
2. 更新提示词模板（`config/prompts/entity_extraction.txt`）
3. 测试新类型的提取效果

### 添加新的关系类型

1. 修改`entity_extractor.py`中的`RELATION_TYPES`列表
2. 更新提示词模板
3. 确保关系类型符合Neo4j命名规范（大写字母+下划线）

## 注意事项

### 连接池管理

- 默认连接池大小为50
- 根据并发需求调整`max_connection_pool_size`
- 避免创建过多存储实例

### 事务边界

- 读操作使用`execute_read`
- 写操作使用`execute_write`
- 不要在事务中执行长时间操作

### 异步上下文

- 异步方法必须在async函数中调用
- 使用`await`关键字等待异步操作
- Windows环境需要设置事件循环策略

### 资源释放

- 使用完毕后调用`storage.close()`
- 避免资源泄漏
- 使用上下文管理器自动释放资源

### LLM幻觉检测

- 实体提取器内置幻觉检测机制
- 检测模板占位符（`{{variable}}`）
- 检测引导式回复（"请提供"、"我将为您"）
- 检测元描述（"以下是提取结果"）

### 性能优化

- 批量操作优于单条操作
- 使用异步方法提高并发性能
- 合理设置分块大小（`max_chunk_chars`）
- 使用MERGE避免重复创建实体
