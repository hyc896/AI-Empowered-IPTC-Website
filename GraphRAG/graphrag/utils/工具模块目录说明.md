# 工具模块目录说明

## 目录作用

utils目录包含GraphRAG的辅助工具模块，提供实体归一化、图数据格式转换、配置加载等通用功能。

## 文件列表

- `__init__.py` - 模块初始化文件
- `entity_normalizer.py` - 实体归一化工具，提供名称归一化、相似度计算、重复检测
- `graph_formatter.py` - 图数据格式转换工具，支持Neo4j到ECharts/D3.js格式转换
- `config_loader.py` - 配置加载器，支持YAML配置和环境变量替换

## 技术说明

### 实体归一化器（entity_normalizer.py）

**核心功能**：
- 名称归一化（去除空格、统一大小写、简繁转换）
- 相似度计算（RapidFuzz Jaro-Winkler距离）
- 重复实体检测（阈值0.9）
- 实体合并和别名映射

**关键方法**：
- `normalize_name(name)` - 归一化实体名称
- `calculate_similarity(name1, name2)` - 计算相似度
- `find_duplicates(entities)` - 查找重复实体
- `merge_entities(entities)` - 合并重复实体

### 图数据格式转换器（graph_formatter.py）

**核心功能**：
- Neo4j数据转换为ECharts Graph格式
- Neo4j数据转换为D3.js Force Graph格式
- 节点大小计算（基于mention_count对数缩放）
- 节点分类和颜色映射
- 关系标签中文化

**关键方法**：
- `to_echarts_format(nodes, links)` - 转换为ECharts格式
- `to_d3_format(nodes, links)` - 转换为D3.js格式
- `calculate_node_size(mention_count)` - 计算节点大小
- `get_node_color(entity_type)` - 获取节点颜色

### 配置加载器（config_loader.py）

**核心功能**：
- YAML配置文件加载
- 环境变量替换（支持`${VAR:default}`语法）
- 配置验证
- 默认值处理

**关键方法**：
- `load_config(config_path)` - 加载配置文件
- `_replace_env_vars(config)` - 递归替换环境变量
- `validate_config(config)` - 验证配置完整性

## 使用示例

### 实体归一化

```python
from graphrag.utils.entity_normalizer import EntityNormalizer

normalizer = EntityNormalizer()

# 归一化名称
normalized = normalizer.normalize_name("  OpenAI  ")
print(normalized)  # "openai"

# 计算相似度
similarity = normalizer.calculate_similarity("OpenAI", "Open AI")
print(similarity)  # 0.95

# 查找重复实体
entities = [
    {"name": "OpenAI", "type": "Company"},
    {"name": "Open AI", "type": "Company"},
    {"name": "Microsoft", "type": "Company"}
]
duplicates = normalizer.find_duplicates(entities)
print(duplicates)  # [("OpenAI", "Open AI")]
```

### 图数据格式转换

```python
from graphrag.utils.graph_formatter import GraphFormatter

formatter = GraphFormatter()

# Neo4j查询结果
nodes = [
    {"id": "1", "name": "OpenAI", "type": "Company", "mention_count": 10}
]
links = [
    {"source": "1", "target": "2", "type": "INVESTS_IN"}
]

# 转换为ECharts格式
echarts_data = formatter.to_echarts_format(nodes, links)
print(echarts_data)
```

### 配置加载

```python
from graphrag.utils.config_loader import ConfigLoader

# 加载配置
config = ConfigLoader.load_config("config/default_config.yaml")

# 验证配置
is_valid = ConfigLoader.validate_config(config)
if not is_valid:
    print("配置无效")
```

## 注意事项

### 归一化策略

- 归一化会丢失原始格式信息
- 保存原始名称到aliases
- 相似度阈值可调整（默认0.9）

### 格式兼容性

- ECharts格式适用于Web可视化
- D3.js格式适用于自定义可视化
- 确保前端库版本兼容

### 配置验证

- 启动前验证配置完整性
- 使用环境变量管理敏感信息
- 提供清晰的错误提示

### 性能考虑

- 相似度计算使用RapidFuzz（高性能）
- 批量归一化优于单条归一化
- 缓存归一化结果避免重复计算

### 环境变量语法

- `${VAR}` - 必需的环境变量
- `${VAR:default}` - 可选的环境变量，提供默认值
- 支持嵌套替换
