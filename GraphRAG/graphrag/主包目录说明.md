# GraphRAG主包目录说明

## 目录作用

graphrag是GraphRAG工具包的主包目录，包含所有核心功能模块。本目录提供知识图谱构建、实体提取、关系推理等核心功能。

## 文件列表

- `__init__.py` - 包初始化文件，导出核心API
- `exceptions.py` - 自定义异常类定义
- `core/` - 核心功能模块（Neo4j存储、实体提取、图谱构建、图谱查询）
- `utils/` - 工具模块（实体归一化、图数据格式转换、配置加载）
- `llm/` - LLM集成模块（支持OpenAI、Anthropic、自定义LLM）
- `schemas/` - 数据模型定义（使用Pydantic）

## 技术说明

### 架构设计

GraphRAG采用模块化设计，各模块职责清晰：

1. **core模块**：核心业务逻辑，包括存储、提取、构建、查询
2. **utils模块**：辅助工具，提供通用功能
3. **llm模块**：LLM集成层，支持多种提供商
4. **schemas模块**：数据模型，使用Pydantic进行验证

### 依赖关系

- core模块依赖llm模块（实体提取需要LLM）
- core模块依赖utils模块（使用工具函数）
- 所有模块使用schemas定义的数据模型

### 异步编程

GraphRAG全面支持异步操作：
- 使用`async/await`语法
- Neo4j操作通过ThreadPoolExecutor实现异步
- LLM调用支持异步和同步两种模式

## 使用示例

### 基础使用

```python
from graphrag.core.storage import Neo4jStorage
from graphrag.core.entity_extractor import EntityExtractor
from graphrag.core.graph_builder import GraphBuilder
from graphrag.llm.openai_client import OpenAIClient

# 初始化存储
storage = Neo4jStorage()
storage.initialize(config['neo4j'])

# 初始化LLM客户端
llm_client = OpenAIClient(api_key="your_key")

# 初始化提取器和构建器
extractor = EntityExtractor(llm_client)
builder = GraphBuilder(storage)

# 提取实体
result = await extractor.extract_entities(text)

# 构建图谱
stats = await builder.add_document_with_entities(
    doc_id="doc_001",
    doc_data=doc_data,
    entities=result['entities'],
    relations=result['relations']
)
```

## 扩展方式

### 添加新的实体类型

在`core/entity_extractor.py`中修改`ENTITY_TYPES`列表：

```python
ENTITY_TYPES = [
    "Company",
    "Person",
    "Technology",
    "Product",
    "Organization",
    "Event",
    "Concept",
    "Location",
    "YourNewType"  # 添加新类型
]
```

### 添加新的关系类型

在`core/entity_extractor.py`中修改`RELATION_TYPES`列表：

```python
RELATION_TYPES = [
    "WORKS_AT",
    "INVESTS_IN",
    "DEVELOPS",
    "COMPETES_WITH",
    "PARTNERS_WITH",
    "BASED_ON",
    "LOCATED_IN",
    "YOUR_NEW_RELATION"  # 添加新关系
]
```

### 添加新的LLM提供商

1. 在`llm/`目录创建新文件（如`your_llm_client.py`）
2. 继承`BaseLLMClient`基类
3. 实现`generate_async`和`generate`方法

## 注意事项

### 资源管理

- 使用完毕后必须调用`storage.close()`关闭连接
- LLM客户端会自动管理连接池
- 避免创建过多的存储实例

### 错误处理

- 所有异常继承自`GraphRAGException`
- 使用try-except捕获特定异常
- 查看日志获取详细错误信息

### 性能优化

- 批量操作优于单条操作
- 使用异步方法提高并发性能
- 合理配置Neo4j连接池大小

### 配置管理

- 使用环境变量管理敏感信息
- 配置文件支持`${VAR:default}`语法
- 启动前验证配置完整性
