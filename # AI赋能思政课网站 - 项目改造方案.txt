# AI赋能思政课网站 - 项目改造方案

> **基于Message Platform项目改造**  
> **目标**：构建思政知识库与案例网站，支持知识图谱展示与智能问答

---

## 📋 项目需求概述

### 新项目核心功能
1. **思政知识库**
   - 基于教材章节的知识体系
   - 知识点结构化存储
   - 智能问答与检索

2. **案例网站**
   - 案例更新档案库
   - 按教材章节分类
   - 案例标签与关联

3. **知识图谱**
   - Neo4j图数据库存储知识关系
   - 可视化展示知识体系
   - 支持关系查询与推理

4. **多源数据整合**
   - 支持案例数据导入
   - 实时更新机制
   - 数据审核流程

### 技术栈要求
- ✅ **前端**：Vue3（保持）
- ✅ **后端**：Python/FastAPI（保持）
- ✅ **关系数据库**：MySQL（保持）
- 🆕 **图数据库**：Neo4j（新增）
- ❓ **向量数据库**：ChromaDB（可选保留，用于语义检索）

**数据库表设计**（MySQL）：
```sql
-- 教材章节表
CREATE TABLE chapters (
    id VARCHAR(36) PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    chapter_number INT NOT NULL,
    description TEXT,
    created_at DATETIME,
    updated_at DATETIME
);

-- 教材小节表
CREATE TABLE sections (
    id VARCHAR(36) PRIMARY KEY,
    chapter_id VARCHAR(36),
    title VARCHAR(200) NOT NULL,
    section_number INT NOT NULL,
    content TEXT,
    created_at DATETIME,
    FOREIGN KEY (chapter_id) REFERENCES chapters(id)
);

-- 知识点表
CREATE TABLE knowledge_points (
    id VARCHAR(36) PRIMARY KEY,
    section_id VARCHAR(36),
    title VARCHAR(200) NOT NULL,
    content TEXT,
    keywords TEXT,
    created_at DATETIME,
    FOREIGN KEY (section_id) REFERENCES sections(id)
);

-- 案例表
CREATE TABLE cases (
    id VARCHAR(36) PRIMARY KEY,
    title VARCHAR(500) NOT NULL,
    content TEXT NOT NULL,
    summary TEXT,
    source_id VARCHAR(36),
    chapter_id VARCHAR(36),
    tags TEXT,
    published_at DATETIME,
    created_at DATETIME,
    updated_at DATETIME,
    FOREIGN KEY (chapter_id) REFERENCES chapters(id)
);

-- 案例来源表
CREATE TABLE case_sources (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    url VARCHAR(500),
    is_active TINYINT(1) DEFAULT 1,
    created_at DATETIME
);

-- 标签表
CREATE TABLE tags (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    category VARCHAR(50),
    created_at DATETIME
);

-- 案例-知识点关联表
CREATE TABLE case_knowledge_relations (
    id VARCHAR(36) PRIMARY KEY,
    case_id VARCHAR(36),
    knowledge_point_id VARCHAR(36),
    relation_type VARCHAR(50),
    created_at DATETIME,
    FOREIGN KEY (case_id) REFERENCES cases(id),
    FOREIGN KEY (knowledge_point_id) REFERENCES knowledge_points(id)
);
```

**流程图**：
```
用户问题
  ↓
向量检索（ChromaDB）→ 相关知识点Top5
  ↓
图谱查询（Neo4j）→ 知识关系图
  ↓
上下文组装 → Prompt构建
  ↓
LLM生成答案（Chat模型）
  ↓
答案+知识来源+案例引用
```

## ⚠️ 风险与注意事项

### 技术风险
1. **Neo4j学习曲线**
   - 风险：Cypher查询语言学习成本
   - 缓解：参考官方文档，从简单查询开始

2. **图谱可视化性能**
   - 风险：大规模图谱渲染卡顿
   - 缓解：分层加载、限制节点数量、使用Canvas渲染

3. **LLM问答准确性**
   - 风险：RAG检索不准确、答案幻觉
   - 缓解：优化检索策略、增加引用来源、人工审核

### 数据风险
1. **数据质量**
   - 风险：案例数据不完整、格式不统一
   - 缓解：数据导入验证、人工审核流程

2. **知识关系复杂性**
   - 风险：知识点关系定义不清晰
   - 缓解：制定知识图谱Schema规范

### 项目风险
1. **需求变更**
   - 风险：教材内容更新频繁
   - 缓解：设计灵活的数据结构

2. **性能优化**
   - 风险：图谱查询性能不佳
   - 缓解：Neo4j索引优化、查询语句优化

## 💡 总结

### 核心改造思路
1. **保留基础框架**：FastAPI + Vue3 + MySQL + LLM
2. **删除采集模块**：不需要自动采集新闻
3. **新增图数据库**：Neo4j支持知识图谱
4. **重构业务逻辑**：从消息管理转向知识管理
5. **增强AI能力**：RAG问答、案例分析

### 技术亮点
- ✨ **双数据库架构**：MySQL存储结构化数据 + Neo4j存储关系图谱
- ✨ **三重检索**：全文检索 + 向量检索 + 图谱检索
- ✨ **智能问答**：基于RAG的知识问答系统
- ✨ **可视化图谱**：直观展示思政知识体系
