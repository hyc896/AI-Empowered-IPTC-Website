## 🚀 快速启动

### 测试模式（无需 Neo4j）

适合测试 PDF 上传和文本提取功能：

```bash
cd backend
python -m uvicorn app.main_test:app --reload --port 8000
```

访问：http://localhost:8000/docs

### 完整模式（需要 Neo4j）

包含完整的实体提取和图谱构建功能：

#### 1. 启动 Neo4j

```bash
# 使用 Docker
docker run -d \
  --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/your_password \
  neo4j:latest
```

#### 2. 配置 GraphRAG

编辑 `GraphRAG/config/default_config.yaml`：

```yaml
neo4j:
  uri: bolt://localhost:7687
  user: neo4j
  password: your_password

llm:
  provider: openai
  model: gpt-4
  api_key: your_openai_api_key
```

#### 3. 启动完整服务

```bash
cd backend
python -m uvicorn app.main:app --reload --port 8000
```

## 📖 API 使用示例

### 1. 上传 PDF 文件

```bash
curl -X POST "http://localhost:8000/api/upload" \
  -F "file=@example.pdf" \
  -F "page_start=1" \
  -F "page_end=20"
```

响应：
```json
{
  "file_id": "uuid-string",
  "filename": "example.pdf",
  "total_pages": 50,
  "current_range": "1-20",
  "text": "提取的文本内容...",
  "char_count": 5000
}
```

### 2. 提取实体（需要完整模式）

```bash
curl -X POST "http://localhost:8000/api/extract" \
  -H "Content-Type: application/json" \
  -d '{
    "file_id": "uuid-string",
    "text": "OpenAI发布了GPT-4模型...",
    "language": "zh",
    "page_range": "1-20"
  }'
```

### 3. 获取图谱数据（需要完整模式）

```bash
curl "http://localhost:8000/api/graph/visualize/uuid-string?page_range=1-20"
```

## 🔧 故障排除

### 问题：服务启动失败

**原因**：Neo4j 未配置或连接失败

**解决**：
1. 使用测试模式：`app.main_test:app`
2. 或配置 Neo4j 连接信息

### 问题：PDF 上传失败

**检查**：
1. 文件大小是否超过 50MB
2. 文件格式是否为 PDF
3. uploads 目录是否有写权限
