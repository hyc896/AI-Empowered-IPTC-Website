# 前端目录说明

## 目录作用

frontend 目录包含 GraphRAG Visualizer 的前端应用代码，基于 Vue 3 + TypeScript + Vite 框架实现。

## 文件列表

### 核心配置文件

- `package.json` - 项目依赖和脚本配置
- `vite.config.ts` - Vite 构建工具配置，包含开发服务器和代理设置
- `tsconfig.json` - TypeScript 编译配置
- `tsconfig.node.json` - Node.js 环境的 TypeScript 配置
- `index.html` - HTML 入口文件

### src/ 目录

#### 主应用文件
- `main.ts` - 应用入口，初始化 Vue 应用和 Element Plus
- `App.vue` - 主应用组件，包含整体布局和状态管理
- `vite-env.d.ts` - Vite 环境类型声明

#### components/ 目录（UI 组件）
- `FileUploader.vue` - PDF 文件上传组件
  - 支持拖拽上传
  - 文件格式和大小验证
  - 页面范围选择（起始页-结束页）
  - 上传进度显示

- `TextPreview.vue` - 文本预览组件
  - 显示提取的 PDF 文本内容
  - 支持滚动查看
  - 字符数统计显示

#### api/ 目录（API 调用）
- `index.ts` - API 封装
  - `uploadPDF()` - 上传 PDF 文件
  - `extractEntities()` - 提取实体和关系
  - Axios 实例配置（baseURL、timeout）

#### types/ 目录（类型定义）
- `index.ts` - TypeScript 类型定义
  - `UploadResponse` - 文件上传响应类型
  - `Entity` - 实体类型
  - `Relation` - 关系类型
  - `ExtractRequest` - 实体提取请求类型
  - `ExtractResponse` - 实体提取响应类型

#### utils/ 目录（工具函数）
- 预留目录，用于存放工具函数（如 LocalStorage 封装等）

## 技术说明

### 核心技术栈
- **Vue 3** - 使用 Composition API 和 `<script setup>` 语法
- **TypeScript** - 类型安全的 JavaScript 超集
- **Vite** - 快速的前端构建工具
- **Element Plus** - Vue 3 UI 组件库
- **Axios** - HTTP 请求库
- **ECharts** - 图表可视化库（待集成）

### 设计模式
- **组件化开发** - 每个功能模块独立为组件
- **响应式数据** - 使用 Vue 3 的 ref 和 reactive
- **类型安全** - 全面使用 TypeScript 类型定义
- **代理配置** - Vite 代理转发 `/api` 请求到后端

### 开发服务器配置
- **端口**: 5173
- **代理**: `/api` -> `http://localhost:8000`
- **热更新**: 支持模块热替换（HMR）

## 使用示例

### 启动开发服务器

```bash
cd frontend
npm install
npm run dev
```

访问：http://localhost:5173

### 构建生产版本

```bash
npm run build
```

构建产物输出到 `dist/` 目录

### 预览生产构建

```bash
npm run preview
```

## 组件使用说明

### FileUploader 组件

```vue
<FileUploader @upload-success="handleUploadSuccess" />
```

**事件**：
- `upload-success` - 上传成功时触发，返回 `UploadResponse` 数据

### TextPreview 组件

```vue
<TextPreview :text="uploadedText" :char-count="charCount" />
```

**Props**：
- `text` (string) - 要显示的文本内容
- `charCount` (number, 可选) - 字符数统计

## 扩展方式

### 添加新组件

1. 在 `src/components/` 创建新的 `.vue` 文件
2. 使用 `<script setup lang="ts">` 语法
3. 定义 Props 和 Emits 类型
4. 在 `App.vue` 或其他组件中导入使用

### 添加新的 API 接口

1. 在 `src/types/index.ts` 定义请求和响应类型
2. 在 `src/api/index.ts` 添加 API 函数
3. 在组件中导入并调用

### 添加新的类型定义

1. 在 `src/types/index.ts` 中添加新的 interface 或 type
2. 使用 `export` 导出供其他模块使用

## 注意事项

### 开发环境
- 确保后端服务运行在 http://localhost:8000
- 前端开发服务器会自动代理 `/api` 请求到后端
- 支持热模块替换，修改代码后自动刷新

### 文件上传
- 仅支持 PDF 格式
- 文件大小限制 50MB
- 默认页面范围 1-20 页

### 错误处理
- 使用 Element Plus 的 `ElMessage` 显示提示信息
- API 错误会显示后端返回的错误详情
- 文件验证失败会显示相应的错误提示

### 性能优化
- 使用 Vite 的按需加载
- Element Plus 组件按需导入（待优化）
- 图片和静态资源优化

## 开发规范

1. **组件命名** - 使用 PascalCase（如 `FileUploader.vue`）
2. **文件头注释** - 每个文件开头添加功能说明注释
3. **类型定义** - 所有 Props、Emits、API 响应都要定义类型
4. **代码风格** - 使用 2 空格缩进，单引号字符串
5. **提交规范** - 遵循项目全局 Git 提交规范
