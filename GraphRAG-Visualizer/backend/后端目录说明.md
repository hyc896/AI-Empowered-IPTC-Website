# 后端目录说明

## 目录作用

backend 目录包含 GraphRAG Visualizer 的后端服务代码，基于 FastAPI 框架实现。

## 文件列表

### 核心文件

- `requirements.txt` - Python 依赖包列表
- `README.md` - 后端服务使用说明

### app/ 目录

#### 主应用文件
- `main.py` - FastAPI 主应用，包含应用初始化、CORS 配置、路由注册
- `config.py` - 配置管理，使用 Pydantic Settings 管理环境变量和配置

#### routers/ 目录（API 路由）
- `upload.py` - 文件上传路由，处理 PDF 上传和文本提取
- `extract.py` - 实体提取路由，调用 GraphRAG 提取实体和关系
- `graph.py` - 图谱查询路由，返回 ECharts 格式的可视化数据

#### services/ 目录（业务逻辑）
- `pdf_service.py` - PDF 处理服务，使用 pdfplumber 提取文本
- `graphrag_service.py` - GraphRAG 服务封装，单例模式管理 GraphRAG 实例

#### models/ 目录（数据模型）
- `schemas.py` - Pydantic 数据模型定义，包含请求和响应的数据结构

## 技术说明

### 核心技术栈
- **FastAPI**: 现代化的 Python Web 框架，支持异步处理
- **pdfplumber**: PDF 文本提取库
- **Pydantic**: 数据验证和设置管理
- **GraphRAG**: 知识图谱构建工具包

### 设计模式
- **单例模式**: GraphRAGService 使用单例模式，确保全局只有一个实例
- **依赖注入**: 使用 FastAPI 的依赖注入系统
- **异步处理**: 所有 I/O 操作使用异步方式

### API 设计
- RESTful 风格
- 统一的错误处理
- 自动生成 OpenAPI 文档

## 使用示例

### 启动服务

```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8000
```

### 访问 API 文档

- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## 扩展方式

### 添加新的 API 端点

1. 在 `app/routers/` 创建新的路由文件
2. 定义路由处理函数
3. 在 `app/main.py` 中注册路由

### 添加新的服务

1. 在 `app/services/` 创建新的服务文件
2. 实现服务类和方法
3. 在路由中调用服务

### 添加新的数据模型

1. 在 `app/models/schemas.py` 中定义 Pydantic 模型
2. 在路由中使用模型进行数据验证

## 注意事项

### 环境配置
- 确保 Neo4j 数据库正常运行
- 配置正确的 GraphRAG 配置文件路径
- 设置 OpenAI API Key

### 文件管理
- 上传的文件保存在 `uploads/` 目录
- 建议定期清理临时文件

### 错误处理
- 所有异常都会被捕获并返回标准错误响应
- 使用 HTTPException 返回 HTTP 错误

### 性能优化
- 使用异步处理提高并发性能
- 分页处理大型 PDF 文件
- GraphRAG 服务使用单例模式避免重复初始化
