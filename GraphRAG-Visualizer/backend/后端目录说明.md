# 后端目录说明

## 目录作用

backend 目录包含 GraphRAG Visualizer 的后端服务代码，基于 FastAPI 框架实现。

## 文件列表

### 根目录文件

- `requirements.txt` - Python 依赖包列表
- `start.py` - 启动脚本，简化服务启动流程
- `README.md` - 后端服务使用说明
- `.env` - 环境变量配置文件
- `后端目录说明.md` - 本文件

### 目录结构

- `app/` - 应用核心代码
- `uploads/` - 上传文件存储目录
- `storage/` - 数据存储目录

### app/ 目录

#### 主应用文件

- `main.py` - FastAPI 主应用，包含应用初始化、CORS 配置、路由注册
- `config.py` - 配置管理，使用 Pydantic Settings 管理环境变量和配置
- `docs_config.py` - API 文档配置

#### routers/ 目录（API 路由）

- `upload.py` - 文件上传路由，处理 PDF/TXT 上传和文本提取
- `extract.py` - 实体提取路由，调用 GraphRAG 提取实体和关系
- `graph.py` - 图谱查询路由，返回 Cosma 格式的可视化数据
- `config.py` - 配置管理路由，动态配置 API 密钥和模型参数

#### services/ 目录（业务逻辑）

- `pdf_service.py` - PDF 处理服务，使用 pdfplumber 提取文本
- `graphrag_service.py` - GraphRAG 服务封装，单例模式管理 GraphRAG 实例
- `progress_manager.py` - 进度管理服务，实时推送处理进度

#### models/ 目录（数据模型）

- `schemas.py` - Pydantic 数据模型定义，包含请求和响应的数据结构

## 技术说明

### 核心技术栈

- **FastAPI**: 现代化的 Python Web 框架，支持异步处理和自动文档生成
- **pdfplumber**: PDF 文本提取库，支持分页处理
- **Pydantic**: 数据验证和设置管理，提供类型安全
- **GraphRAG**: 知识图谱构建工具包，负责实体提取和关系识别
- **python-dotenv**: 环境变量管理
- **aiofiles**: 异步文件操作
- **python-slugify**: 文本规范化处理

### 设计模式

- **单例模式**: GraphRAGService 使用单例模式，确保全局只有一个实例
- **依赖注入**: 使用 FastAPI 的依赖注入系统
- **异步处理**: 所有 I/O 操作使用异步方式
- **进度管理**: 使用 ProgressManager 实时推送处理进度

### API 设计

- RESTful 风格的接口设计
- 统一的错误处理机制
- 自动生成 OpenAPI 文档（Swagger UI）
- 支持 CORS 跨域请求
- 返回 Cosma 格式的图谱数据

## 使用示例

### 启动服务

```bash
cd backend
pip install -r requirements.txt

# 配置环境变量（创建 .env 文件）
# GRAPHRAG_CONFIG_PATH=../GraphRAG/config/default_config.yaml

# 使用启动脚本
python start.py

# 或使用 uvicorn 直接启动
python -m uvicorn app.main:app --reload --port 8000
```

### 访问 API 文档

- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

### API 调用示例

```bash
# 上传文件
curl -X POST "http://localhost:8000/api/upload" \
  -F "file=@test.pdf" \
  -F "page_start=1" \
  -F "page_end=10"

# 获取图谱数据
curl "http://localhost:8000/api/graph/cosma/{file_id}"

# 删除图谱
curl -X DELETE "http://localhost:8000/api/graph/{file_id}"
```

## 扩展方式

### 添加新的 API 端点

1. 在 `app/routers/` 创建新的路由文件
2. 定义路由处理函数
3. 在 `app/main.py` 中注册路由

### 添加新的服务

1. 在 `app/services/` 创建新的服务文件
2. 实现服务类和方法
3. 在路由中调用服务

### 添加新的数据模型

1. 在 `app/models/schemas.py` 中定义 Pydantic 模型
2. 在路由中使用模型进行数据验证

## 注意事项

### 环境配置

- 确保 Neo4j 数据库正常运行
- 配置正确的 GraphRAG 配置文件路径
- 在 `.env` 文件中设置环境变量
- 支持通过 API 动态配置 LLM 参数

### 文件管理

- 上传的文件保存在 `uploads/` 目录
- 图谱数据保存在 `storage/` 目录
- 建议定期清理临时文件
- 支持 PDF 和 TXT 格式

### 错误处理

- 所有异常都会被捕获并返回标准错误响应
- 使用 HTTPException 返回 HTTP 错误
- 详细的错误日志记录
- 友好的错误提示信息

### 性能优化

- 使用异步处理提高并发性能
- 分页处理大型 PDF 文件
- GraphRAG 服务使用单例模式避免重复初始化
- 进度管理器实时推送处理状态

### 安全性

- CORS 配置限制跨域访问
- 文件类型验证
- 文件大小限制
- 路径遍历防护
