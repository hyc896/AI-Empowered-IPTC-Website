# 前端项目目录说明

## 目录作用

frontend-new 是 GraphRAG Visualizer 的前端应用，基于 Vue 3 + TypeScript + AntV G6 构建，提供知识图谱可视化界面。

## 文件列表

### 根目录文件

- `package.json` - 项目依赖和脚本配置
- `vite.config.ts` - Vite 构建工具配置
- `tsconfig.json` - TypeScript 编译配置
- `tsconfig.node.json` - Node 环境 TypeScript 配置
- `index.html` - HTML 入口文件
- `.gitignore` - Git 忽略文件配置
- `.env.example` - 环境变量示例
- `README.md` - 项目说明文档

### src/ 目录

#### api/ - API 服务层
- `index.ts` - Axios 实例配置和拦截器
- `upload.ts` - 文件上传 API
- `extract.ts` - 实体提取 API
- `graph.ts` - 图谱查询 API

#### types/ - TypeScript 类型定义
- `index.ts` - 基础类型（Entity、Relation）
- `api.ts` - API 相关类型
- `graph.ts` - G6 图谱类型

#### components/ - 组件目录
- `Layout/MainLayout.vue` - 主布局组件
- `Upload/UploadDialog.vue` - 文件上传对话框
- `Graph/GraphViewer.vue` - G6 图谱可视化组件
- `Entity/EntityList.vue` - 实体列表组件

#### views/ - 页面组件
- `Home.vue` - 主页面，整合所有功能模块

#### router/ - 路由配置
- `index.ts` - Vue Router 路由配置

#### 应用入口
- `App.vue` - 根组件
- `main.ts` - 应用入口文件

## 技术说明

### 核心技术栈
- Vue 3 - 使用 Composition API
- TypeScript - 完整类型支持
- Vite - 快速开发构建
- Element Plus - UI 组件库
- AntV G6 - 图可视化引擎

### 架构设计
- 组件化开发，职责清晰
- API 层统一管理后端接口
- TypeScript 类型定义完整
- 响应式布局设计

## 使用示例

### 安装依赖
```bash
npm install
```

### 启动开发服务器
```bash
npm run dev
```

### 构建生产版本
```bash
npm run build
```

## 扩展方式

### 添加新页面
1. 在 `src/views/` 创建页面组件
2. 在 `src/router/index.ts` 添加路由配置

### 添加新组件
1. 在 `src/components/` 对应目录创建组件
2. 在需要的地方导入使用

### 添加新 API
1. 在 `src/api/` 创建 API 文件
2. 在 `src/types/` 定义类型
3. 在组件中调用

## 注意事项

### G6 使用注意
- 使用 `toRaw()` 处理响应式数据
- 组件卸载时销毁 G6 实例
- 避免频繁重新渲染

### 性能优化
- 使用 `shallowRef` 存储大数据
- 按需加载组件
- 合理使用计算属性

### 开发规范
- 组件文件头添加注释说明
- 使用 TypeScript 类型定义
- 遵循 Vue 3 最佳实践
