# 前端目录说明

## 目录作用

frontend 目录包含 GraphRAG Visualizer 的前端应用代码，基于 Vue 3 + TypeScript + Vite 构建。

## 文件列表

### 根目录文件

- `package.json` - 项目依赖和脚本配置
- `package-lock.json` - 依赖版本锁定文件
- `vite.config.ts` - Vite 构建工具配置
- `tsconfig.json` - TypeScript 编译配置
- `tsconfig.node.json` - Node.js 环境的 TypeScript 配置
- `index.html` - HTML 入口文件
- `.env.development` - 开发环境变量配置
- `.env.production` - 生产环境变量配置
- `.gitignore` - Git 忽略文件配置

### src/ 目录

#### 核心文件
- `main.ts` - 应用入口文件，创建 Vue 应用实例
- `App.vue` - 根组件，包含主要应用逻辑和布局
- `style.css` - 全局样式文件
- `vite-env.d.ts` - Vite 环境类型声明

#### api/ 目录（API 接口层）
- `index.ts` - Axios 实例配置和通用 API 配置
- `upload.ts` - 文件上传相关接口
- `graph.ts` - 图谱数据查询接口

#### components/ 目录（Vue 组件）
- `CosmaViewer.vue` - Cosma 图谱查看器组件，核心可视化组件
- `Dialog/` - 对话框组件目录
  - `ConfirmDialog.vue` - 确认对话框组件
  - `InputDialog.vue` - 输入对话框组件
  - `SettingsDialog.vue` - 设置对话框组件
- `Progress/` - 进度显示组件目录

#### cosma/ 目录（Cosma 图谱引擎）
- `graph.ts` - Cosma 图谱渲染引擎，基于 D3.js 和 Graphology

#### types/ 目录（TypeScript 类型定义）
- `cosma.ts` - Cosma 相关类型定义
- `index.ts` - 通用类型定义（待补充）

#### utils/ 目录（工具函数）
- 工具函数和辅助方法（待补充）

## 技术说明

### 核心技术栈

- **Vue 3**: 使用 Composition API 和 `<script setup>` 语法
- **TypeScript**: 提供类型安全和更好的开发体验
- **Vite**: 快速的开发服务器和构建工具
- **D3.js**: 强大的数据可视化库，用于图谱渲染
- **Graphology**: 图数据结构库，管理节点和边
- **Fuse.js**: 模糊搜索库，实现实体搜索功能
- **Axios**: HTTP 客户端，处理 API 请求
- **Marked**: Markdown 解析器，用于文本渲染

### 设计模式

- **组件化设计**: 将功能拆分为独立的 Vue 组件
- **响应式状态管理**: 使用 Vue 3 的 ref 和 reactive 管理状态
- **API 层分离**: 将 API 调用逻辑与组件逻辑分离
- **类型安全**: 使用 TypeScript 接口定义数据结构

### Cosma 图谱引擎

Cosma 是一个基于 D3.js 的力导向图可视化引擎，特点：
- 支持节点拖拽和缩放
- 力导向布局算法
- 节点和边的交互
- 搜索和筛选功能
- 自定义样式和主题

## 使用示例

### 启动开发服务器

```bash
cd frontend
npm install
npm run dev
```

### 构建生产版本

```bash
npm run build
npm run preview
```

### 环境变量配置

在 `.env.development` 或 `.env.production` 中配置：

```env
VITE_API_BASE_URL=http://localhost:8000
```

## 扩展方式

### 添加新的 API 接口

1. 在 `src/api/` 目录创建新的接口文件
2. 使用 `src/api/index.ts` 中导出的 axios 实例
3. 定义接口函数并导出

示例：

```typescript
import api from './index'

export const fetchData = async () => {
  const response = await api.get('/api/data')
  return response.data
}
```

### 添加新的 Vue 组件

1. 在 `src/components/` 目录创建新的 `.vue` 文件
2. 使用 `<script setup>` 语法和 TypeScript
3. 在需要的地方导入并使用组件

示例：

```vue
<script setup lang="ts">
import { ref } from 'vue'

const count = ref(0)
</script>

<template>
  <div>{{ count }}</div>
</template>
```

### 添加新的类型定义

1. 在 `src/types/` 目录创建或编辑类型文件
2. 定义 TypeScript 接口或类型
3. 导出供其他模块使用

示例：

```typescript
export interface User {
  id: string
  name: string
  email: string
}
```

## 注意事项

### 开发规范

- 使用 TypeScript 编写代码，确保类型安全
- 遵循 Vue 3 Composition API 最佳实践
- 组件命名使用 PascalCase（如 `MyComponent.vue`）
- 文件命名使用 camelCase（如 `myUtils.ts`）
- 每个文件头部添加注释说明文件作用

### 代码风格

- 使用 2 空格缩进
- 使用单引号而非双引号
- 组件 props 和 emits 必须定义类型
- 避免使用 any 类型
- 合理使用 ref 和 reactive

### 性能优化

- 大型列表使用虚拟滚动
- 合理使用 computed 缓存计算结果
- 避免在模板中使用复杂表达式
- 图谱渲染使用 requestAnimationFrame
- 大数据量使用分页或懒加载

### 调试技巧

- 使用 Vue DevTools 调试组件状态
- 使用浏览器开发者工具查看网络请求
- 使用 console.log 或 debugger 调试代码
- 检查 Vite 控制台输出的错误信息
