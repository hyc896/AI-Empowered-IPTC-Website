# 组件目录说明

## 目录作用

components 目录包含所有可复用的 Vue 组件，负责实现应用的各个功能模块。

## 文件列表

### 核心组件

- `CosmaViewer.vue` - Cosma 图谱查看器组件
  - 核心可视化组件
  - 集成 Cosma 图谱引擎
  - 支持节点拖拽、缩放、搜索
  - 提供图谱交互功能
  - 管理图谱状态和数据

### Dialog/ 目录（对话框组件）

- `ConfirmDialog.vue` - 确认对话框组件
  - 用于确认删除等操作
  - 支持自定义标题和内容
  - 提供确认和取消按钮
  - 使用 Promise 返回用户选择

- `InputDialog.vue` - 输入对话框组件
  - 用于重命名等输入操作
  - 支持单行文本输入
  - 输入验证功能
  - 支持占位符和默认值

- `SettingsDialog.vue` - 设置对话框组件
  - 配置 API 密钥和基础 URL
  - 选择 LLM 模型
  - 设置提取参数
  - 保存配置到本地存储

### Progress/ 目录（进度组件）

- 进度显示相关组件
- 实时显示处理进度
- 支持多阶段进度展示

## 技术说明

### 组件设计原则

- 单一职责：每个组件只负责一个功能
- 可复用性：组件设计考虑复用场景
- 类型安全：使用 TypeScript 定义 props 和 emits
- 响应式：使用 Vue 3 Composition API

### CosmaViewer 组件架构

- 使用 D3.js 进行图谱渲染
- 使用 Graphology 管理图数据结构
- 使用 Fuse.js 实现模糊搜索
- 支持力导向布局算法
- 提供丰富的交互功能

## 扩展方式

### 添加新组件

1. 在 `components/` 目录创建新的 `.vue` 文件
2. 使用 `<script setup lang="ts">` 语法
3. 定义 props 和 emits 的类型
4. 实现组件逻辑和模板

示例：

```vue
<script setup lang="ts">
interface Props {
  title: string
  visible: boolean
}

const props = defineProps<Props>()
const emit = defineEmits<{
  close: []
}>()
</script>

<template>
  <div v-if="visible">
    <h2>{{ title }}</h2>
    <button @click="emit('close')">关闭</button>
  </div>
</template>
```

## 注意事项

### 组件开发规范

- 组件文件名使用 PascalCase（如 `MyComponent.vue`）
- 每个组件文件头部添加注释说明组件作用
- Props 和 emits 必须定义类型
- 避免在组件中直接调用 API，应通过 props 传递数据
- 组件应该是纯粹的，避免副作用

### 性能优化

- 大型组件考虑拆分为多个小组件
- 使用 `v-show` 而非 `v-if` 处理频繁切换的元素
- 避免在模板中使用复杂计算，使用 computed
- 图谱组件使用防抖和节流优化交互性能

### 可访问性

- 为交互元素添加适当的 ARIA 属性
- 确保键盘可访问性
- 提供合适的焦点管理
