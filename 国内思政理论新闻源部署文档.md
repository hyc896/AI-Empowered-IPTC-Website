## 二、采集器配置详情

每个采集器均按照建议配置：

```json
{
  "region": "中国",
  "language": "zh",
  "max_articles": 20,
  "interval": 3600,
  "mysql_table": "mp_{source_name}_messages",
  "collector_module": "backend.sources.{source_name}.collector",
  "chroma_collection": "{source_name}"
}
```

### 关键特性：

1. **三阶段架构**：抓取 → 预处理（字段丰富化）→ 存储
2. **无需翻译**：源内容已是中文，跳过translation步骤
3. **字段丰富化**：自动提取 region、industry_tags、ai_tag
4. **智能去重**：基于URL和external_id
5. **向量存储**：自动同步到ChromaDB
6. **错误处理**：完善的异常捕获和日志记录

## 三、如何启动和测试

### 1. 启动消息平台

```bash
cd "d:\AI-Empowered IPTC Website\message_platform"
conda activate personal_agent
python start.py
```

访问 API 文档：http://localhost:11528/docs

### 2. 启动 Celery 定时任务（自动采集）

```bash
conda activate personal_agent
celery -A backend.celery_app worker --loglevel=info --pool=solo
```

### 3. 手动测试单个采集器

```python
# 在Python环境中测试
from backend.sources.people_theory.collector import PeopleTheoryCollector
from backend.database.connection import get_mysql_session

config = {
    'id': 'test-source-id',
    'mysql_table': 'mp_people_theory_messages',
    'chroma_collection': 'people_theory',
    'config': {
        'region': '中国',
        'language': 'zh',
        'max_articles': 5
    }
}

collector = PeopleTheoryCollector(config)
result = collector.collect()
print(result)
```

### 4. 查看采集结果


## 七、问题排查

### 1. 采集器无法启动

**可能原因**：
- 依赖包未安装（playwright、beautifulsoup4等）
- 浏览器驱动未安装

**解决方案**：
```bash
pip install playwright beautifulsoup4 lxml
playwright install chromium
```

### 2. 无法抓取到文章

**可能原因**：
- 网站结构变化
- CSS选择器不匹配
- 网站访问限制

**解决方案**：
- 手动访问网站，检查结构
- 使用浏览器开发者工具查看元素选择器
- 调整采集器代码中的选择器

### 3. 字段丰富化失败

**可能原因**：
- LLM API配置错误
- API额度不足
- 网络连接问题

**解决方案**：
- 检查 `.env` 中的LLM配置
- 查看API使用情况
- 检查网络连接
