# 案例生成逻辑调整说明

## 一、数据结构变更概述

### 1.1 新增字段

knowledge_points.json中每个知识点新增了4个字段：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| section | string | 节序号（一、二、三等） | "一" |
| section_title | string | 节标题 | "马克思主义中国化时代化的提出" |
| subsection | string | 小节标题（如果有） | "领导权问题" |
| teaching_application | string | 教学应用说明 | "本案例主要适用于导论第一节马克思主义中国化时代化的提出部分的辅助教学" |

### 1.2 字段用途

- **section + section_title**: 精确定位知识点在教材中的位置（章节级别）
- **subsection**: 进一步细化到小节（如果教材有小节划分）
- **teaching_application**: 为生成的案例提供标准化的教学应用说明

## 二、案例生成逻辑调整方案

### 2.1 当前问题

原有的案例生成逻辑可能只使用了 `chapter` 和 `chapter_id` 字段，无法精确到具体的节和小节，导致生成的案例缺乏精确的教材定位信息。

### 2.2 调整建议

#### 方案A：在案例生成时直接使用 teaching_application 字段

**优点**：最简单，无需修改提示词逻辑
**实现**：直接将 `teaching_application` 字段的值作为案例的教学应用说明

```python
# 伪代码示例
def generate_case(knowledge_point):
    case = {
        "name": knowledge_point["name"],
        "theory_description": knowledge_point["theory_description"],
        # ... 其他字段
        "teaching_application": knowledge_point["teaching_application"]  # 直接使用
    }
    return case
```

#### 方案B：动态构建教学应用说明

**优点**：更灵活，可以根据不同场景定制说明文字
**实现**：根据 section、section_title、subsection 动态构建

```python
# 伪代码示例
def build_teaching_application(knowledge_point):
    chapter = knowledge_point["chapter"]
    section = knowledge_point["section"]
    section_title = knowledge_point["section_title"]
    subsection = knowledge_point["subsection"]

    # 提取章节号（如"第二章"）
    chapter_num = chapter.split()[0] if chapter else ""

    # 构建基础说明
    if section and section_title:
        base_text = f"{chapter_num}第{section}节{section_title}"
    else:
        base_text = chapter

    # 如果有小节，添加小节信息
    if subsection:
        base_text += f"中{subsection}"

    return f"本案例主要适用于{base_text}部分的辅助教学"
```

### 2.3 推荐方案

**推荐使用方案A**，理由：
1. teaching_application 字段已经预先生成，格式统一
2. 减少运行时计算，提高性能
3. 避免因动态构建导致的格式不一致问题

```python
# 修改后
def generate_case_from_knowledge_point(kp):
    return {
        "name": kp["name"],
        "theory_description": kp["theory_description"],
        "application_scenarios": kp["application_scenarios"],
        "typical_keywords": kp["typical_keywords"],
        "chapter": kp["chapter"],

        "teaching_application": kp.get("teaching_application", "")
    }
```
