# 图书馆网站爬虫使用说明

## 目标网站
https://mfindecupl.libsp.cn/#/Home

## 脚本文件

### 1. analyze_libsp.py - 网站分析脚本
用于分析网站结构和API接口

**功能:**
- 使用Selenium捕获网络请求，查找API接口
- 测试常见的API端点
- 分析页面元素结构

**使用方法:**
```bash
python scripts/analyze_libsp.py
```

### 2. crawl_libsp.py - 书籍爬虫脚本
用于爬取首页书籍信息

**功能:**
- 自动访问网站并爬取书籍信息
- 支持翻页爬取多页数据
- 导出为CSV和JSON格式

**使用方法:**
```bash
python scripts/crawl_libsp.py
```

### 3. search_libsp.py - 搜索爬虫脚本（推荐）
用于搜索并爬取指定关键词的书籍

**功能:**
- 支持关键词搜索
- 自动滚动加载更多结果
- 导出为CSV和JSON格式

**使用方法:**
```bash
python scripts/search_libsp.py
# 然后输入搜索关键词，如：人工智能、法学等
```

**示例:**
```bash
# 搜索"人工智能"相关书籍
echo "人工智能" | python scripts/search_libsp.py
```

1. 确保已安装Chrome浏览器

3. 将ChromeDriver添加到系统PATH

## 爬取策略

### 方法1: 页面解析（当前实现）
- 使用Selenium加载完整页面
- 解析HTML提取书籍信息
- 适用于前端渲染的SPA应用

### 方法2: API接口（推荐）
如果通过分析脚本找到了API接口，可以直接调用API：

```python
import requests

# 示例：假设找到了API接口
api_url = "https://mfindecupl.libsp.cn/api/books"
response = requests.get(api_url)
books = response.json()
```

## 注意事项

1. **反爬虫机制**
   - 网站可能有访问频率限制
   - 建议在请求间添加延时（已在代码中实现）
   - 使用合理的User-Agent

2. **登录要求**
   - 如果网站需要登录才能访问，需要先手动登录获取Cookie
   - 可以使用Selenium自动登录或手动提供Cookie

3. **数据合规**
   - 仅用于学习和研究目的
   - 遵守网站的robots.txt规则
   - 不要过度请求造成服务器压力

4. **调试建议**
   - 首次运行时注释掉`--headless`参数，观察浏览器行为
   - 使用浏览器开发者工具（F12）查看网络请求
   - 检查页面实际的HTML结构和class名称

## 扩展功能

### 添加登录功能
```python
def login(driver, username, password):
    driver.get(f"{self.base_url}/#/login")
    driver.find_element(By.ID, "username").send_keys(username)
    driver.find_element(By.ID, "password").send_keys(password)
    driver.find_element(By.CSS_SELECTOR, "button[type='submit']").click()
    time.sleep(2)
```

### 添加详情页爬取
```python
def crawl_book_detail(self, book_url):
    driver.get(book_url)
    # 提取详细信息
    description = driver.find_element(By.CLASS_NAME, "description").text
    return description
```

## 故障排查

1. **ChromeDriver版本不匹配**
   - 确保ChromeDriver版本与Chrome浏览器版本匹配

2. **元素未找到**
   - 增加等待时间
   - 检查选择器是否正确
   - 使用浏览器开发者工具验证元素

3. **网络超时**
   - 检查网络连接
   - 增加timeout参数
   - 使用代理（如需要）
