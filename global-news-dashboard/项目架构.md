# Global News Dashboard 前端项目架构文档

## 一、项目概述

Global News Dashboard是message_platform的Web前端界面，提供消息源管理和消息查看功能。

**核心功能**：
- 消息源管理：创建、编辑、删除、启用/禁用消息源
- 消息列表展示：按来源、关键词筛选消息
- 语义搜索：基于ChromaDB的智能检索
- 统计信息展示：消息总数、来源统计等

**技术栈**：
- Vue 3：渐进式JavaScript框架
- TypeScript：类型安全的JavaScript超集
- Vite：下一代前端构建工具
- Element Plus：Vue 3 UI组件库
- Pinia：Vue状态管理库
- Vue Router：Vue官方路由
- Axios：HTTP客户端
- Markdown-it：Markdown渲染
- Highlight.js：代码高亮
- SCSS：CSS预处理器

**运行环境**：
- Node.js 16+
- npm 或 yarn
- 浏览器：Chrome/Edge/Firefox最新版

## 三、核心模块说明

### 3.1 API接口层（src/api/）

**index.ts**：
- Axios实例配置
- 请求/响应拦截器
- 错误处理
- 超时配置（30秒）
- Base URL配置（http://localhost:11528）

**message.ts**：
- getMessageSources()：获取消息源列表
- getMessageSource(sourceId)：获取单个消息源
- createMessageSource(data)：创建消息源
- updateMessageSource(sourceId, data)：更新消息源
- deleteMessageSource(sourceId)：删除消息源
- activateMessageSource(sourceId)：启用消息源
- deactivateMessageSource(sourceId)：禁用消息源
- getMessageSourceStatus(sourceId)：获取消息源状态
- getExternalMessages(params)：获取消息列表
- searchMessages(request)：语义搜索消息
- getMessageStats()：获取统计信息
- getAvailableSources()：获取可用消息源类型

### 3.2 类型定义层（src/types/）

**models.ts**：
- MessageSource：消息源数据模型
- ExternalMessage：消息数据模型
- MessageStats：统计信息模型

**api.ts**：
- PaginatedResponse<T>：分页响应模型
- PaginationParams：分页参数模型

**store.ts**：
- MessageState：消息Store状态类型
- MessageSourceState：消息源Store状态类型

### 3.3 状态管理层（src/stores/）

**message.ts**：
- 状态：
  - messages：消息列表
  - sources：消息源列表
  - currentSource：当前选中的消息源
  - stats：统计信息
  - loading：加载状态
- 操作：
  - fetchMessages()：获取消息列表
  - fetchSources()：获取消息源列表
  - createSource()：创建消息源
  - updateSource()：更新消息源
  - deleteSource()：删除消息源
  - searchMessages()：搜索消息
  - fetchStats()：获取统计信息

### 3.4 路由层（src/router/）

**index.ts**：
- /：默认路由，重定向到消息列表
- /messages：消息列表页
- /message-sources：消息源管理页

路由模式：History模式

### 3.5 视图层（src/views/）

**Messages/index.vue**：
- 消息列表展示
- 消息筛选（按来源、关键词）
- 消息搜索
- 分页功能
- 消息详情查看

**MessageSources/index.vue**：
- 消息源列表展示
- 新建消息源表单
- 编辑消息源
- 删除消息源
- 启用/禁用消息源
- 消息源状态展示

### 3.6 组件层（src/components/）

**common/ExternalMessageCard.vue**：
- 消息卡片展示
- 属性：message（ExternalMessage）
- 显示：标题、内容、发布时间、来源、相似度

**common/MessageSourceCard.vue**：
- 消息源卡片展示
- 属性：source（MessageSource）
- 显示：名称、类别、状态、最后抓取时间
- 操作：编辑、删除、启用/禁用

**common/MarkdownRenderer.vue**：
- Markdown内容渲染
- 属性：content（string）
- 功能：代码高亮、链接处理

### 3.7 工具函数层（src/utils/）

**date.ts**：
- formatDate(date)：格式化日期
- formatRelativeTime(date)：相对时间格式化
- parseDate(str)：解析日期字符串

**format.ts**：
- truncateText(text, length)：截断文本
- highlightKeyword(text, keyword)：关键词高亮
- formatNumber(num)：格式化数字

**markdown.ts**：
- renderMarkdown(content)：渲染Markdown
- sanitizeHtml(html)：HTML清理

### 3.8 常量定义（src/constants/）

**index.ts**：
- API_TIMEOUT：API超时时间（30000ms）
- PAGE_SIZE：分页大小（20）
- MESSAGE_SOURCE_CATEGORIES：消息源类别映射

## 五、API集成

### 5.1 后端API地址

- 开发环境：http://localhost:11528
- 生产环境：通过环境变量VITE_API_BASE_URL配置

### 5.2 API路径规范

所有API请求均带有 `/api/v1` 前缀

### 5.3 请求/响应处理

- 请求拦截器：添加通用请求头
- 响应拦截器：统一错误处理、消息提示
- 错误处理：HTTP错误、网络错误、业务错误

### 5.4 Vite代理配置

开发环境下，`/api` 请求自动代理到 `http://localhost:11528`

配置位置：vite.config.ts

```typescript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:11528',
      changeOrigin: true
    }
  }
}
```

## 十一、与后端集成

### 11.1 API对接

前端调用后端RESTful API，所有端点遵循 /api/v1 规范

### 11.2 数据流向

```
用户操作 → Vue组件 → Pinia Store → API调用 → 后端服务
                                              ↓
                                          响应数据
                                              ↓
页面更新 ← 组件响应 ← Store更新 ← API响应 ← 后端返回
```

### 11.3 错误处理

- HTTP 4xx：客户端错误，显示错误提示
- HTTP 5xx：服务器错误，显示服务异常
- 网络错误：显示网络连接失败


## 十三、开发规范

### 13.1 命名规范

- 组件：PascalCase（MessageSourceCard.vue）
- 文件夹：kebab-case（message-sources/）
- 函数/变量：camelCase（fetchMessages）
- 常量：UPPER_SNAKE_CASE（API_TIMEOUT）

### 13.2 TypeScript规范

- 所有API函数必须定义返回类型
- 所有Props必须定义类型
- 避免使用any，优先使用unknown
- 使用接口定义数据模型

### 13.3 代码注释

- 函数必须添加注释说明
- 复杂逻辑必须添加注释
- API调用必须注释用途

### 13.4 Git提交规范

- feat：新功能
- fix：修复bug
- refactor：重构
- docs：文档更新
- style：样式调整
- test：测试相关
- chore：构建/工具相关
